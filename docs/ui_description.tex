\section{Introduction}
% ============================================================

This document describes the user interface (UI) of the \textbf{IoT-based Smart Parking Management System (IoT-SPMS)}, a web application developed for Ho Chi Minh City University of Technology (HCMUT). The system is built using \textbf{Next.js} (React) with \textbf{TypeScript} and styled with \textbf{Tailwind CSS}. UI components are provided by the \textbf{shadcn/ui} component library.

The application serves multiple user roles:
\begin{itemize}
    \item \textbf{End Users} (Learners, Faculty, Staff) -- view parking availability, manage sessions, and handle billing.
    \item \textbf{Parking Operators} -- manage visitor tickets and monitor active sessions.
    \item \textbf{System Administrators} -- full access including user management, pricing policies, and analytics reports.
\end{itemize}

The UI adopts a \textbf{role-based access control} (RBAC) model, where the navigation sidebar and available pages dynamically adapt based on the authenticated user's role.

\section{Authentication -- Login Page}
\label{sec:login}
% ============================================================

The login page is the entry point of the application. It replicates the visual appearance of the \textbf{HCMUT Central Authentication Service (CAS)}, providing a familiar single sign-on (SSO) experience for university members.

\subsection{Layout and Components}

The page is organized as follows:

\begin{itemize}
    \item \textbf{Header Bar}: A dark purple banner (\texttt{\#210F7A}) displaying the HCMUT logo and the title ``Central Authentication Service''.
    \item \textbf{Login Form (Left Panel)}: A light gray card containing:
    \begin{itemize}
        \item Username and Password input fields with a pale yellow background (\texttt{\#FFFFDD}).
        \item A ``Warn me before logging me into other sites'' checkbox.
        \item ``Login'' and ``Clear'' action buttons styled with a blue gradient.
        \item A ``Change password?'' hyperlink.
        \item Error messages displayed in a red dotted-border box when authentication fails.
    \end{itemize}
    \item \textbf{Information Sidebar (Right Panel)}: Contains:
    \begin{itemize}
        \item Language selection links (Vietnamese / English).
        \item A ``Please note'' section explaining the SSO mechanism.
        \item Technical support contact information.
        \item A ``Demo Credentials'' section listing available test accounts dynamically loaded from the \texttt{hcmut\_sso.json} data file.
    \end{itemize}
    \item \textbf{Footer}: Copyright notice and a link to the Jasig CAS project.
\end{itemize}

\section{Dashboard Layout}
\label{sec:dashboard-layout}
% ============================================================

Upon successful authentication, users are redirected to the dashboard. The dashboard layout is consistent across all internal pages and consists of three main regions.

\subsection{Top Navigation Header}

A sticky header bar with a dark blue background (\texttt{\#003366}) containing:
\begin{itemize}
    \item \textbf{Application Branding}: A circular white icon with a car symbol, the title ``Smart Parking System'', and the subtitle ``HCMUT IoT-SPMS''.
    \item \textbf{Notification Bell}: A bell icon with a red dot indicator. Clicking it opens a dropdown showing recent system notifications (e.g., ``System Sync Completed'', ``Zone C is Full'').
    \item \textbf{User Profile Menu}: Displays the user's avatar (initials), full name, and email. Clicking opens a dropdown with the user's role badge, a link to Settings, and a Logout button.
    \item \textbf{Hamburger Menu} (mobile only): Toggles the sidebar on small screens.
\end{itemize}

\subsection{Left Sidebar Navigation}

A white sidebar (264px wide) with navigation links that adapt based on the user's role:

\begin{itemize}
    \item \textbf{End Users}: Dashboard, Parking Map, My Sessions, Billing.
    \item \textbf{Operators}: Dashboard, Parking Map, All Sessions, Visitor Tickets.
    \item \textbf{Administrators}: Dashboard, Parking Map, All Sessions, User Management, Pricing Policies, Reports, Visitor Tickets.
\end{itemize}

The active page is highlighted with a dark blue background. At the bottom of the sidebar, a user info card displays the user's name, student ID or department, and current debt amount (if any).

On mobile devices, the sidebar overlays the content with a semi-transparent black backdrop.

\subsection{Main Content Area}

The main content area occupies the remaining space to the right of the sidebar. It has a light gray background (\texttt{\#F9FAFB}) with 24px padding.

\begin{itemize}
    \item Available slots (green).
    \item Occupied slots (red).
    \item Total slots (blue).
    \item Overall occupancy percentage (purple).
\end{itemize}

\subsection{Legend}

A horizontal bar with color-coded indicators: Available (green), Occupied (red), Sensor Issue (yellow), and Your Vehicle (blue).

\subsection{Zone Overview}

A 4-column grid of clickable zone cards. Each card shows:
\begin{itemize}
    \item Zone name and status badge (Available / Nearly Full / Full).
    \item Number of available slots out of total.
    \item Supported vehicle types (motorbike, car) as outline badges.

\end{itemize}

Clicking a zone card reveals the \textbf{Detailed Slot Map} -- a 10-column grid where each cell represents a parking slot. Slots are color-coded: green for available (showing slot number), red for occupied (showing a car icon), and yellow with a warning icon for sensor issues. Hovering over a slot displays a tooltip with details (status, vehicle plate if occupied).

\subsection{Electronic Guidance Signage}

A simulation of the physical LED signage boards at parking entry points (UC-RTM-03). Each zone is displayed as a dark panel with a large monospace number showing available slots (colored green, yellow, or red), or ``FULL'' in red text.

\section{Pricing Policies (Admin Only)}
\label{sec:pricing}
% ============================================================

The Pricing Policies page (UC-ADM-04) allows administrators to configure parking fee structures.

\subsection{Policy Cards}

Policies are displayed as a grid of cards (3 columns). Each card shows:
\begin{itemize}
    \item Policy name and an active/inactive toggle switch.
    \item User role and vehicle type as outline badges.
    \item Billing type badge (Accumulated -- blue, Immediate -- orange, Exempt -- green).
    \item Rate per hour, flat rate, and maximum daily rate in VND.
    \item Billing cycle (monthly).
    \item An ``Edit Policy'' button.
\end{itemize}

Inactive policies are displayed with reduced opacity (60\%).

\subsection{Add / Edit Policy Dialog}

Modal dialogs for creating or editing a policy contain form fields:
\begin{itemize}
    \item Policy Name, User Role, Vehicle Type, Billing Type (dropdowns).
    \item Rate/Hour, Flat Rate, and Max Daily Rate (numeric inputs in VND).
    \item An active/inactive toggle.
\end{itemize}

\subsection{Fee Calculation Engine}

An informational card explaining how parking fees are calculated for:
\begin{itemize}
    \item \textbf{Accumulated Billing} (Learners/Staff): Entry scan $\to$ exit duration calculation $\to$ fee added to debt $\to$ monthly BKPay billing.
    \item \textbf{Immediate Payment} (Visitors): Ticket at entry $\to$ ticket scan at exit $\to$ fee calculated $\to$ immediate cash/QR payment.
\end{itemize}

\section{Responsive Design}
\label{sec:responsive}
% ============================================================

The application is fully responsive across devices:

\begin{itemize}
    \item \textbf{Desktop} ($\geq$1024px): Full sidebar navigation is always visible alongside the main content. Grid layouts use 4 columns for statistics cards.
    \item \textbf{Tablet} (768px--1023px): Sidebar collapses and can be toggled via the hamburger menu. Grid layouts typically reduce to 2 columns.
    \item \textbf{Mobile} ($<$768px): Sidebar overlays the content with a dark backdrop. Grid layouts stack into single columns. The user's name and email in the header are hidden, showing only the avatar.
\end{itemize}

Tables use horizontal scrolling (\texttt{overflow-x-auto}) on smaller screens to maintain readability.
